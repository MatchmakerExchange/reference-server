The Exchange for Matchmakers

Summary

  1. Make an authenticated MME API request to this service
  2. If you don't want to query all services, you can specify a subset
  3. Receive a combined response, containing matches from all services

Features

  1. Authenticates the requesting service
  2. Validates requests and responses conform to the the MME API jsonschema specification
  3. Normalizes request ontologies:
    • maps gene symbols and Entrez IDs to Ensembl IDs
    • resolves HPO synonyms
  4. Enables new services to be queried by the entire MME with only a single key exchange

Usage

Make a request to a specific server: pc

curl -X POST \
    -H 'X-Auth-Token: <your-exchange-server-auth-token>' \
    -H 'Content-Type: application/vnd.ga4gh.matchmaker.v1.0+json' \
    -H 'Accept: application/vnd.ga4gh.matchmaker.v1.0+json' \
    -d '{
          "patient": {
            "id":"1",
            "contact": {"name": "Jane Doe", "href": "mailto:jdoe@example.edu"},
            "features": [{"id": "HP:0000522"}],
            "genomicFeatures": [{"gene": {"id": "NGLY1"}}],
            "test": true
          }
        }' \
    https://x.matchmakerexchange.org/v1/servers/pc/match
